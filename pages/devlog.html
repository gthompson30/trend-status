Project Description:

TrendStatus uses Google Trends Data to analyze trend data of a random topic and assign it a medical status, for example: terminally ill, alive and well, hospitalized, six feet under, etc. The website allows you to search up a topic and gives you the medical status of that topic.

OBSTACLES OVERCOME

  • At first, we tried to scrape Google Trends data from the requests module. We were unable to get requests working, because the filename was "requests.py", and this caused a "circular import" error, as the Python file was trying to import itself.

    • This was eventually resolved naming the Python file something other than "requests", although we ended up using a dedicated module, instead of requests

  • After trying and failing to use requests to scrape Google Trends data, we tried using multiple different Python modules, many of which were unusable because they spit out cryptic error messages we were unable to understand, required a Google account, or were otherwise confusing.

    • We eventually found the Python library, pytrends, which allowed us to easily retrieve Google Trends data, with no cryptic error messages, and no requirement to sign up for a Google account.

  • After we were able to load in data from Google Trends, we were initially unsure how to scrape the data for historical popularity, because the Google Trends data that pytrends retrieved was a variable type that was custom to the module.

    • We eventually found that we could take the retrieved data, take the index of the trend we wanted to look at, and convert this to a list. The resulting list containing the raw data of the historical popularity, which we could analyze to calculate the appropriate medical condition

  • We were initially unsure of how to analyze the popularity data and calculate a "positivity" or "negativity" to the data. We at first thought that we could compare the current data to the highest point, and make the score based off of that.

    • We went through multiple different ideas for calculating the positivity of the graph, but eventually decided to make a list containing the slope between each item of the list. We then calculated the average of this list, and this average gave a pretty good numeric representation of the positivity of the graph.

  • We originally didn't know how to make boxes to contain elements in HTML

    • We figured out that we could use div tags to contain various other elements, and have backgrounds or separate properties for those elements.

  • When setting up main.py, we were unable to import our trends.py file (which we had originally coded to calculate popularity of a searched topic). Thus, we ended up moving the function in the file (status('term')) to the main.py file. However, that was when we found out that it was the pytrends module we were unable to import

    • At first, Gabriel Thompson created a web server on his machine to process status('term') instead and our code would just request the results from that.

    • Eventually, Kevin Wang helped us and we were able to import the pytrends module by moving the downloaded pytrends module to our final project folder instead of the local library.

LESSONS LEARNED

  • Make sure the name of the file you are working on is not the name of a module you may need.

  • If you need to download a module to make a certain function work but the code needs to be able to be run by everyone, move the downloaded module folder to the same folder as your work so it can be accessed.

  • Shadows are very picky. Here's a short summary of factors to consider: the color of your text, the color of your shadow (making all shadows black is both tacky and unrealistic), how offset your shadow is from your test, how blurred your shadow is, etc. The best way to make a good shadow is a dark -light combination : light text with dark shadow or the opposite. Never not offset the shadow and just blur because that leads to tackiness (unless you blur it very far and that's the effect you are looking for.) Also, color combinations with the rest of your page also matter. Some colors clash and having colors that are too close makes the page look off. The best way is just to mess around with the shadows a lot but these are the basic rules to consider.
