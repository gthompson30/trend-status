About TrendStatus

TrendStatus uses Google Trends Data to analyze trend data of a random topic and assign it a medical status, for example: terminally ill, alive and well, hospitalized, six feet under, etc. The website allows you to search up a topic and gives you the medical status of that topic.

The front-end for the website is done with HTML and CSS, while the back-end is done with Python CGI. The Google Trends data is retreived through the pytrends module, which we use to get the historical data for any search term typed into the website. We then look at the current popularity for a trend, relative to its highest ever popularity, to assign the trend a numerical score representing its "health". Our program then assigns the trend a "health description" based on its numerical score. The program then generates a webpage with the aforementioned information, which is displayed in the browser.

The health descriptions are assigned according to the following ranges:

Description Popularity Range
Better than ever 100% (only)
Alive and well! 75% - 99%
Consistently Healthy 30% - 74%
Had a small accident, but not too bad 15% - 29%
Hospitalized, but possible recovery 5% - 14%
Terminally ill 2% - 4%
Six feet under 0% - 1%

The "current trends", as shown on the right were retrieved through the requests module, by sending GET requests to the Google Trends website, and parsing the HTML content to calculate the top trends.
